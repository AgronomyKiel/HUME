Description

Defines core data types and classes for representing state variables, parameters, and external variables in a crop simulation model. Provides a standardized way to handle variables that change over time (states), fixed model parameters, and variables imported from other modules or the environment.

Overview

Classes, Interfaces, Objects and Records

Name Description
Class TExternV TExternV is a class representing an external variable in the model. Values of model elements outside a submodel are called "external variables". They can be made accessible by adressing a pointer for that value. The default method for setting the link to the external variable is to search for it in the model by its name. It extends THumeNumEntity and provides methods for getting and setting the value, as well as creating the external variable with a name, units, and an external value type.
Class THumeEntity THumeEntity is the base class for all objects representing either numerical values (states, variables, parameters, constants) or Options. If the destination is a console app it is derived from TObject. For the use of within the IDE/GUI applications it is derived from the TPersistent class.
Class THumeNumEntity THumeNumEntity is a base class for numerical entities in the model, such as variables and external values. It extends THumeEntity with properties for numerical values, units, and flags for plotting and output.
Class TOption TOption is a class representing an option with a string value and a list of possible options.
Class TPar TPar is a class representing a parameter variable in the model, which can be selected for optimization or sensitivity analysis. It extends TVar and adds properties for maximum and minimum expected values, error, and selection flags.
Class TState TState is a class representing a state variable in the model, which has an initial value and a rate of change. It extends TVar and provides a constructor for creating state variables with initial value, change rate, and comment.
Class TVar TVar is a class representing a variable, i.e. a value that can be calculated for each time step in the model from other variables, state variables or parameters. It extends THumeNumEntity and provides methods for self writing of source code for its initialization, rate calculation.
